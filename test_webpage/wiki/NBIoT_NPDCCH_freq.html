1. Overview

The Narrowband Physical Downlink Control Channel (NPDCCH) is an NB-IoT specific downlink control channel defined in 3GPP Release 13. It carries Downlink Control Information (DCI) used to schedule NB-IoT UEs on both the downlink (NPDSCH) and uplink (NPUSCH).

NPDCCH is transmitted on the anchor carrier and operates within a single 180 kHz narrowband resource block. Unlike LTE PDCCH which uses CCEs, NPDCCH uses Narrowband Control Channel Elements (NCCEs).

2. Operation Modes

NB-IoT and NPDCCH support three deployment operation modes:

  • In-band: Deployed within an existing LTE carrier, occupying one of the LTE resource blocks. Guard band resource blocks cannot be used.
  • Guard-band: Deployed in the guard band of an LTE carrier.
  • Stand-alone: Deployed in a dedicated 200 kHz channel, e.g., refarmed GSM spectrum.

The operation mode affects which subframes are available for NPDCCH transmission due to LTE reference signal collisions in in-band mode.

3. NPDCCH Formats

NPDCCH supports two formats defined by the number of NCCEs used:

FormatNCCEsDCI Formats CarriedUsage
NPDCCH Format 01 NCCEDCI format N0, N1 (1 NCCE)UE-specific search space (USS)
NPDCCH Format 12 NCCEsDCI format N0, N1, N2 (2 NCCEs)Common search space (CSS) and USS

1. Support NPDCCH format 0 and format 1.

DCI Format N0 – Used for NPUSCH scheduling (uplink grant).
DCI Format N1 – Used for NPDSCH scheduling (downlink assignment) and RRC connection setup.
DCI Format N2 – Used for paging and direct indication.

4. Frequency Multiplexing

Each NCCE occupies one subcarrier pair (2 subcarriers × one slot). The 12 subcarriers of the NB-IoT resource block are split into two NCCEs:

  • NCCE 0: Subcarriers 0–5 of the resource block
  • NCCE 1: Subcarriers 6–11 of the resource block

Frequency multiplexing of NPDCCH candidates from different UEs is possible within the same subframe by assigning different NCCEs. This increases the number of simultaneously scheduled UEs per TTI.

Note: In-band deployment may have reduced available NPDCCH capacity due to LTE CRS (Cell-specific Reference Signal) puncturing on certain subcarriers.

5. NCCE Aggregation

NPDCCH supports two aggregation levels:

  • Aggregation Level 1: Uses 1 NCCE (Format 0). Higher spectral efficiency, suitable for good channel conditions (CE Mode A, low repetition).
  • Aggregation Level 2: Uses 2 NCCEs (Format 1). Required for common search space; provides additional coding gain for coverage-limited UEs (CE Mode B).

6. Search Spaces

NPDCCH uses two types of search spaces:

Search SpaceAbbr.PurposeAggregation Level
Common Search SpaceCSSRAR (Random Access Response), paging, SI scheduling2 NCCEs only
UE-specific Search SpaceUSSDedicated data scheduling per UE1 or 2 NCCEs

The UE monitors the CSS during predefined subframes and the USS during dedicated search space subframes configured by RRC signaling.

7. Coverage Enhancement & Repetition

To support extended coverage (CE Mode A and CE Mode B), NPDCCH supports repetition of the same control information across multiple subframes:

  • CE Mode A: Repetitions R ∈ {1, 2, 4, 8, 16, 32, 64, 128} for USS; R ∈ {1, 2, 4, 8} for CSS RAR.
  • CE Mode B: Repetitions R ∈ {1, 2, 4, 8, 16, 32, 64, 128, 256, 512, 1024, 2048} for USS.

The repetition R is applicable to RAR and Msg3 retransmission. Higher repetition values extend the coverage range but reduce the number of available scheduling opportunities per second.

Warning: Excessive repetition values (>128) can cause significant scheduling delays and reduce cell capacity. Tune npdcch-NumRepetitions based on actual UE coverage needs.
