MR7501: Uplink Throughput Test Failure Analysis

From LTE-5G-Wiki Last edited: 11:05, 28 June 2023  |  9 KB (874 words)
  • Article
  • Discussion
  • Read
  • View source
  • View history

During regression testing for MR7501 SP2, a recurring uplink throughput degradation was identified across multiple test cases targeting the 4x4 UL MIMO configuration in TDD band 41 deployments, where peak throughput measurements consistently fell 18–23% below the expected reference value defined in the feature acceptance criteria, prompting a root cause investigation that ultimately traced the failure to an incorrect UE category capability advertisement during the RRC connection setup phase, in which the eNodeB scheduler was assigning transport block sizes optimized for UE Category 12 rather than the Category 16 capability reported by the test UE, a discrepancy introduced by a regression in the capability parsing logic that had been silently present since the MR7499 integration but had not been caught in earlier test cycles due to the limited UE category diversity of the automated test pool maintained by the CI environment at the time of that release.
